Console Logger

In this chapter you may find some interesting information about using the console logging writer. But before continue reading please note, the console logger is split into two different implementations.

The first implementation called Console Logger Standard can be used platform independently. In contrast to that, the second implementation called Console Logger Windows can only be used on Windows platforms. The reason behind, the Console Logger Windows uses the Win32–API directly to be able to support additional Windows features.

Logger Basic Settings

Below can be found a fully qualified example of how to use the console logger settings. You may recognize that the time format, the part split, the quick edit mode as well as the window title have been changed.

Copy
using Plexdata.LogWriter.Abstraction;
using Plexdata.LogWriter.Definitions;
using Plexdata.LogWriter.Extensions;
using Plexdata.LogWriter.Logging.Windows;
using Plexdata.LogWriter.Settings;
using System;

namespace ConsoleLoggerExample
{
    class Program
    {
        static void Main(String[] args)
        {
            IConsoleLoggerSettings settings = new ConsoleLoggerSettings
            {
                LogLevel = LogLevel.Trace,
                ShowTime = true,
                TimeFormat = "G",
                PartSplit = '|',
                QuickEdit = true,
                WindowTitle = "Console Logger Example",
            };

            ILogger logger = new ConsoleLogger(settings);

            logger.Trace("Trace...");
            logger.Debug("Debug...");
            logger.Verbose("Verbose...");
            logger.Message("Message...");
            logger.Warning("Warning...");
            logger.Error("Error...");
            logger.Fatal("Fatal...");
            logger.Critical("Critical...");
        }
    }
}

Logger Color Settings

Changing the console logger’s message coloring can be done by using property Coloring of the console logger settings. How it can be accomplished is shown below.

Copy
static void Main(String[] args)
{
    ...
    settings.Coloring[LogLevel.Verbose] = new Coloring(ConsoleColor.DarkGreen, ConsoleColor.White);
    settings.Coloring[LogLevel.Critical] = new Coloring(ConsoleColor.DarkGray, ConsoleColor.Magenta);

    logger.Trace("Trace...");
    logger.Debug("Debug...");
    logger.Verbose("Verbose...");
    logger.Message("Message...");
    logger.Warning("Warning...");
    logger.Error("Error...");
    logger.Fatal("Fatal...");
    logger.Critical("Critical...");
    ...            
}

Dependency Injection

Dependency injection is always an important issue. Therefore, this section wants to show how to accomplish this task for the console logger implementation.

Dependency Injection in .NET Core

Using dependency injection in a simple .NET Core application might be a bit tricky. Here are the steps to get it running or just click on to download the complete example project.

Step 1

Start with a new .NET Core console application.

Step 2

Install following NuGet packages.

  • Plexdata.LogWriter.Abstraction
  • Plexdata.LogWriter.Console
  • Plexdata.LogWriter.Console.Standard
  • Microsoft.Extensions.Configuration
  • Microsoft.Extensions.Configuration.Abstractions
  • Microsoft.Extensions.Configuration.Binder
  • Microsoft.Extensions.Configuration.Json
  • Microsoft.Extensions.DependencyInjection

Step 3

Now create a configuration file, such as appsettings.json and make sure this file has its properties set to Content, Copy always

Step 4

Add content below to the new configuration file.

Copy
{ 
    "ConsoleLoggerSettings": {
        "LogLevel": "Trace",
        "LogTime": "Local",
        "LogType": "Raw",
        "ShowTime": true,
        "PartSplit": "|",
        "WindowTitle": "Hello World" 
    }
}

Step 5

Create a helper class, for example named ConsoleLoggerConfiguration, that is able to process the configuration content. This helper class might look like shown below.

Copy
public class ConsoleLoggerConfiguration : ConsoleLoggerSettings
{
    public ConsoleLoggerConfiguration(IConfigurationRoot config)
        : base()
    {
        if (config == null)
        {
            throw new ArgumentNullException(nameof(config));
        }

        config.GetSection("ConsoleLoggerSettings").Bind(this);
    }
}

Step 6

Implement a class that gets injected with its logger instance like show as follows.

Copy
public class ExampleClassWithLoggerInjection
{
    private readonly IConsoleLogger logger;

    public ExampleClassWithLoggerInjection(IConsoleLogger logger)
    {
        this.logger = logger ?? throw new ArgumentNullException(nameof(logger));
    }

    public void MethodToCall()
    {
        this.logger.Message("Here I am...");
    }
}

Step 7

In the program’s Main method implement the required dependency injection part as shown below.

Copy
static void Main(String[] args)
{
    IConfigurationBuilder builder = new ConfigurationBuilder();
    builder.SetBasePath(Directory.GetCurrentDirectory()).AddJsonFile("appsettings.json", false);

    IServiceCollection service = new ServiceCollection();
    service.AddSingleton<IConfigurationRoot>(builder.Build());
    service.AddSingleton<IConsoleLoggerSettings, ConsoleLoggerConfiguration>();
    service.AddSingleton<IConsoleLogger, ConsoleLogger>();
    service.AddTransient(typeof(ExampleClassWithLoggerInjection));

    IServiceProvider provider = service.BuildServiceProvider();

    ExampleClassWithLoggerInjection example = provider.GetService<ExampleClassWithLoggerInjection>();
    example.MethodToCall();

    Console.ReadKey();
}

Step 8

Finally, run the program. The console output should look like as shown as here.

Dependency Injection in ASP.NET Core

Using dependency injection in an ASP.NET Core application might be a bit easier. Anyway, here are the steps to get it running or just click on to download the complete example project.

Step 1

Start with a new ASP.NET Core console application and choose API as type in the dialog box. After project creation, open project settings, choose Debug tab, select the project name, for example ConsoleLoggerExample2, from Profile drop–list as well as un–tick the check–box Launch browser. Finally, in the toolbar select the project name, for example ConsoleLoggerExample2, instead of IIS Express.

Step 2

Install following NuGet packages.

  • Plexdata.LogWriter.Abstraction
  • Plexdata.LogWriter.Console
  • Plexdata.LogWriter.Console.Standard

Step 3

Open configuration file appsettings.json for edit and add content below.

Copy
{ 
    ...
    "ConsoleLoggerSettings": {
        "LogLevel": "Trace",
        "LogTime": "Local",
        "LogType": "Raw",
        "ShowTime": true,
        "PartSplit": "|",
        "WindowTitle": "Hello World" 
    }
    ...
}

Step 4

Open file Startup.cs for edit and put following lines into the methods as shown below.

Copy
public void ConfigureServices(IServiceCollection services)
{
    ...
    services.AddSingleton<Plexdata.LogWriter.Abstraction.IConsoleLoggerSettings, Plexdata.LogWriter.Settings.ConsoleLoggerSettings>();
    services.AddSingleton<Plexdata.LogWriter.Abstraction.IConsoleLogger, Plexdata.LogWriter.Logging.Standard.ConsoleLogger>();
    ...
}

public void Configure(IApplicationBuilder app, IHostingEnvironment env)
{
    ...
    this.Configuration.GetSection("ConsoleLoggerSettings").Bind(app.ApplicationServices.GetService<Plexdata.LogWriter.Abstraction.IConsoleLoggerSettings>());
    ...
}

Step 5

As next it would be useful to inject the console logger somewhere, for example in the value controller. For this purpose open file ValuesController.cs and add a constructor and a field as shown as follows.

Copy
private readonly IConsoleLogger logger;

public ValuesController(IConsoleLogger logger)
{
    this.logger = logger ?? throw new ArgumentNullException(nameof(logger));
}

After that, move to method Get() and write a logging message like shown in the next code–snippet.

Copy
public ActionResult<IEnumerable<string>> Get()
{
    this.logger.Message("Here I am...");
    return new string[] { "value1", "value2" };
}

Step 6

Finally, build and run the program. Then open a browser and call URL https://localhost:5001/api/values. After that, the console output should look like as shown as here.
