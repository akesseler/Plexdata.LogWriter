Mail Settings

The mail logger settings are the settings used together with each of the mail logger implementations. These settings are defined in interface IMailLoggerSettings.

  • Conditions
  • Address
  • Username
  • Password
  • SMTP Host
  • SMTP Port
  • Use SSL
  • Encoding
  • Subject
  • Receivers
  • Clear Copies
  • Blind Copies

Conditions

To be able the send logging messages by e–mail it is at least required to define the settings Address and Username and Password and SMTP Host. Additionally, at least one of the receiver addresses (Receivers or Clear Copies or Blind Copies) must be set. Otherwise, sending of e–mails will be impossible.

On the other hand it is important to know that the mail logger does not throw any exception if sending an e–mail has failed. But all exceptions are printed out in the Console. This in turn requires a Console window, to be able to recognize trouble with sending e–mails.

Address

With this setting it becomes possible to define the e–mail sender address. Such a sender address might be an e–mail address that is specially reserved for a logging service. Example are logging@example.org or noreply@example.org. The Address setting is actually the From e–mail address.

Username

This setting represents the account name required to be able to login at the mail server. Depending on the SMTP host login procedure, the username might be an e–mail address. The Username in combination with setting Password is known as Network Credential.

Password

This setting represents the account password required to be able to login at the mail server. The Password in combination with setting Username is known as Network Credential. Be aware, there might be a security issue if such a password is put into an unencrypted configuration file!

SMTP Host

This setting describes the fully qualified name of the SMTP host from which e–mails are sent. Such a host name could be for example mail.example.org.

SMTP Port

This setting allows to define the SMTP port number. Usually, such a port number is set to 587 for new SMTP servers. Legacy SMTP servers may use 25 as SMTP port. Default value is 587.

Use SSL

This setting allows to determine whether the usage of SSL is enabled or not. Default value is enabled.

Encoding

This setting allows to define the used e–mail encoding. This encoding is used for the mail subject as well as for the mail body text. Default value is UTF–8.

Subject

This setting allows to define the e–mail subject. The logging message is used as subject if this setting is not define (null, empty, whitespaces). Default value is empty.

Receivers

This setting allows to define the list of main e–mail recipients (To). Default value is empty.

Clear Copies

This setting allows to define the list of copy e–mail recipients (carbon copy CC). Default value is empty.

Blind Copies

This setting allows to define the list of blind e–mail recipients (blind carbon copy BCC). Default value is empty.
